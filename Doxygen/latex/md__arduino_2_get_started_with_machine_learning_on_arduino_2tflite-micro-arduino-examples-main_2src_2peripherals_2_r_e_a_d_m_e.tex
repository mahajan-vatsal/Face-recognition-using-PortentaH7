The \href{https://www.waveshare.com/wm8960-audio-hat.htm}{\texttt{ Waveshare WM8960 Audio HAT}} utilizes the WM8960 codec. This audio board provides a stereo 3.\+5mm earphone jack and 1W speaker outputs as well as stereo microphone inputs.

The earphone output is best used with 24 ohm (or higher) impedance headphones.

The USB power source must be capable of supplying a minimum of 500 milli-\/amps in order to use the audio board with speakers attached.\hypertarget{md__arduino_2_get_started_with_machine_learning_on_arduino_2tflite-micro-arduino-examples-main_2src_2peripherals_2_r_e_a_d_m_e_autotoc_md44}{}\doxysubsection{\texorpdfstring{Supported Capabilities}{Supported Capabilities}}\label{md__arduino_2_get_started_with_machine_learning_on_arduino_2tflite-micro-arduino-examples-main_2src_2peripherals_2_r_e_a_d_m_e_autotoc_md44}
This audio board has the following supported capabilities\+:
\begin{DoxyItemize}
\item stereo input
\item stereo output
\item earphone output
\item 1W (max) speaker output
\item 8 or 16 bits per channel
\item standard samples rates from 8k\+Hz to 48k\+Hz
\item simultaneous record and playback
\end{DoxyItemize}\hypertarget{md__arduino_2_get_started_with_machine_learning_on_arduino_2tflite-micro-arduino-examples-main_2src_2peripherals_2_r_e_a_d_m_e_autotoc_md45}{}\doxysubsection{\texorpdfstring{Audio Board Pin Connections}{Audio Board Pin Connections}}\label{md__arduino_2_get_started_with_machine_learning_on_arduino_2tflite-micro-arduino-examples-main_2src_2peripherals_2_r_e_a_d_m_e_autotoc_md45}

A generic button event API is provided in the \href{/src/peripherals/button.h}{\texttt{ button.\+h}} header file.

The generic button API provides access to the debounced up/down state of the button as well as the following button events\+:

A generic LED API is provided in the \href{/src/peripherals/led.h}{\texttt{ led.\+h}} header file.

The physical default LED will vary between peripherals and platforms. The GPIO pin that connects to the default LED is defined in the \href{/src/peripherals/peripherals.h}{\texttt{ peripherals.\+h}} header file.

The generic LED API provides the following functionality\+:


\begin{DoxyItemize}
\item Turning the LED on/off
\item Blinking the LED at an application defined duty cycle and rate
