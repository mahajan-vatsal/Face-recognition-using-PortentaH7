\chapter{CMSIS NN Software Library}
\hypertarget{index}{}\label{index}\index{CMSIS NN Software Library@{CMSIS NN Software Library}}
\hypertarget{index_Introduction}{}\doxysection{\texorpdfstring{Introduction}{Introduction}}\label{index_Introduction}
This user manual describes the CMSIS NN software library, a collection of efficient neural network kernels developed to maximize the performance and minimize the memory footprint of neural networks on Arm Cortex-\/M processors.

The library is divided into a number of functions each covering a specific category\+:
\begin{DoxyItemize}
\item \doxylink{group___n_n_conv}{Convolution Functions} Convolution Functions
\item \doxylink{group___acti}{Activation Functions}
\item \doxylink{group___f_c}{Fully-\/connected Layer Functions} Fully-\/connected Layer Functions
\item \doxylink{group___s_v_d_f}{SVDF Functions} Layer Functions
\item \doxylink{group___pooling}{Pooling Functions} Functions
\item \doxylink{group___softmax}{Softmax Functions} Functions
\item \doxylink{group__group_elementwise}{Elementwise Functions} Basic math Functions
\end{DoxyItemize}\hypertarget{index_Processors}{}\doxysection{\texorpdfstring{Supported Processors}{Supported Processors}}\label{index_Processors}
CMSIS-\/\+NN targets Cortex-\/M processors with typically three different implementations for each function. Each targets a different group of processors.
\begin{DoxyItemize}
\item Processors without Single Instruction Multiple Data(\+SIMD) capability (e.\+g, Cortex-\/\+M0)
\item Processors with DSP extension (e.\+g Cortex-\/\+M4)
\item Processors with Arm M-\/\+Profile Vector Extension(\+MVE) instructions (e.\+g Cortex-\/\+M55) The right implementation is picked through feature flags and the user does not have to explicit set it.
\end{DoxyItemize}\hypertarget{index_Framework}{}\doxysection{\texorpdfstring{Quantization Specification}{Quantization Specification}}\label{index_Framework}
The library follows the \href{https://www.tensorflow.org/lite/performance/quantization_spec}{\texttt{ int8}} and int16 quantization specification of Tensor\+Flow Lite for Microcontrollers. \hypertarget{index_Overview}{}\doxysection{\texorpdfstring{Block Diagram}{Block Diagram}}\label{index_Overview}

\hypertarget{index_Feature}{}\doxysubsection{\texorpdfstring{Feature flag based}{Feature flag based}}\label{index_Feature}
The macros below are defined in a build system based on feature flags for a chosen processor or architecture input to a compiler. These tie in to the classification in \doxysectlink{index_Macros}{Pre-\/processor Macros}{1}.

For a CMSIS-\/\+NN file compiled as {\itshape armclang -\/mcpu=cortex-\/m4 --target=arm-\/arm-\/none-\/eabi -\/I$<$\+CMSIS Core Include$>$ -\/Ofast -\/O file.\+c} , ARM\+\_\+\+MATH\+\_\+\+DSP is enabled as Cortex-\/\+M4 has the DSP extension as a feature.


\begin{DoxyItemize}
\item {\ttfamily ARM\+\_\+\+MATH\+\_\+\+DSP} -\/ Selects code for processors with DSP extension.
\item {\ttfamily ARM\+\_\+\+MATH\+\_\+\+MVEI} -\/ Selects code for processors which supports MVE instructions.
\end{DoxyItemize}\hypertarget{index_MiscFlags}{}\doxysubsection{\texorpdfstring{User Set}{User Set}}\label{index_MiscFlags}

\begin{DoxyItemize}
\item {\ttfamily ARM\+\_\+\+MATH\+\_\+\+AUTOVECTORIZE} Applicable when ARM\+\_\+\+MATH\+\_\+\+MVEI is active to let the compiler auto vectorize functions, if available, that uses inline assembly. This has to be explicitly set at compile time.
