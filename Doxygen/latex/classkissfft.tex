Calculates the complex Discrete Fourier Transform.

The size of the passed arrays must be passed in the constructor. The sum of the squares of the absolute values in the {\ttfamily dst} array will be {\ttfamily N} times the sum of the squares of the absolute values in the {\ttfamily src} array, where {\ttfamily N} is the size of the array. In other words, the l\+\_\+2 norm of the resulting array will be {\ttfamily sqrt(\+N)} times as big as the l\+\_\+2 norm of the input array. This is also the case when the inverse flag is set in the constructor. Hence when applying the same transform twice, but with the inverse flag changed the second time, then the result will be equal to the original input times {\ttfamily N}. \Hypertarget{classkissfft_a1f034808a68b2eef02f52145b82abf04}\index{kissfft$<$ scalar\_t $>$@{kissfft$<$ scalar\_t $>$}!transform\_real@{transform\_real}}

Calculates the Discrete Fourier Transform (DFT) of a real input of size {\ttfamily 2\texorpdfstring{$\ast$}{*}N}.

The 0-\/th and N-\/th value of the DFT are real numbers. These are stored in {\ttfamily dst}\mbox{[}0\mbox{]}.\doxylink{ceva__typedef_8h_a5f5a64ef503db0fc824df8320929b126}{real()} and {\ttfamily dst}\mbox{[}1\mbox{]}.\doxylink{ceva__typedef_8h_a06ebc6bb2d8547759c483e78d2087c51}{imag()} respectively. The remaining DFT values up to the index N-\/1 are stored in {\ttfamily dst}\mbox{[}1\mbox{]} to {\ttfamily dst}\mbox{[}N-\/1\mbox{]}. The other half of the DFT values can be calculated from the symmetry relation 
\begin{DoxyCode}{0}
\DoxyCodeLine{DFT(src)[2*N-\/k]\ ==\ conj(\ DFT(src)[k]\ );}

\end{DoxyCode}
 The same scaling factors as in {\ttfamily \doxylink{classkissfft_a166aed860359f5b9250c5fcc899aae14}{transform()}} apply.

\begin{DoxyNote}{Note}
For this to work, the types {\ttfamily scalar\+\_\+t} and {\ttfamily cpx\+\_\+t} must fulfill the following requirements\+:
\end{DoxyNote}
For any object {\ttfamily z} of type {\ttfamily cpx\+\_\+t}, {\ttfamily reinterpret\+\_\+cast$<$scalar\+\_\+t(\&)\mbox{[}2\mbox{]}$>$(z)}\mbox{[}0\mbox{]} is the real part of {\ttfamily z} and {\ttfamily reinterpret\+\_\+cast$<$scalar\+\_\+t(\&)\mbox{[}2\mbox{]}$>$(z)}\mbox{[}1\mbox{]} is the imaginary part of {\ttfamily z}. For any pointer to an element of an array of {\ttfamily cpx\+\_\+t} named {\ttfamily p} and any valid array index {\ttfamily i}, {\ttfamily reinterpret\+\_\+cast$<$\+T\texorpdfstring{$\ast$}{*}$>$(p)}\mbox{[}2\texorpdfstring{$\ast$}{*}i\mbox{]} is the real part of the complex number {\ttfamily p}\mbox{[}i\mbox{]}, and {\ttfamily reinterpret\+\_\+cast$<$\+T\texorpdfstring{$\ast$}{*}$>$(p)}\mbox{[}2\texorpdfstring{$\ast$}{*}i+1\mbox{]} is the imaginary part of the complex number {\ttfamily p}\mbox{[}i\mbox{]}.

Since C++11, these requirements are guaranteed to be satisfied for {\ttfamily scalar\+\_\+ts} being {\ttfamily float}, {\ttfamily double} or {\ttfamily long} {\ttfamily double} together with {\ttfamily cpx\+\_\+t} being {\ttfamily std\+::complex$<$scalar\+\_\+t$>$}.
