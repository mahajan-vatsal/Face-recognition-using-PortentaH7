Our test scripts assume that the non static {\ttfamily user} mode installation of QEMU is available in the PATH. For example, if using QEMU for ARM testing, please make sure {\ttfamily qemu-\/arm} is installed and available to the test scripts.

You can use {\ttfamily ci/install\+\_\+qemu.\+sh} to download, build and install the version of qemu that is used as part of the CI.\hypertarget{md__arduino_2_get_started_with_machine_learning_on_arduino_2tflite-micro-main_2tensorflow_2lite_2micro_2docs_2qemu_autotoc_md222}{}\doxysection{\texorpdfstring{Software Emulation with QEMU}{Software Emulation with QEMU}}\label{md__arduino_2_get_started_with_machine_learning_on_arduino_2tflite-micro-main_2tensorflow_2lite_2micro_2docs_2qemu_autotoc_md222}
Tensor\+Flow Lite Micro makes use of \href{https://qemu.org}{\texttt{ QEMU}} to for testing cross compiled tests.

QEMU can quickly test unit tests that are cross compiled for non x64\+\_\+86 hardware.\hypertarget{md__arduino_2_get_started_with_machine_learning_on_arduino_2tflite-micro-main_2tensorflow_2lite_2micro_2docs_2qemu_autotoc_md223}{}\doxysection{\texorpdfstring{Running Unit Tests}{Running Unit Tests}}\label{md__arduino_2_get_started_with_machine_learning_on_arduino_2tflite-micro-main_2tensorflow_2lite_2micro_2docs_2qemu_autotoc_md223}
All unit tests can be ran using {\ttfamily tensorflow/lite/micro/tools/ci\+\_\+build/test\+\_\+cortex\+\_\+m\+\_\+qemu.\+sh} for the cortex-\/m processor.\hypertarget{md__arduino_2_get_started_with_machine_learning_on_arduino_2tflite-micro-main_2tensorflow_2lite_2micro_2docs_2qemu_autotoc_md224}{}\doxysection{\texorpdfstring{Useful External Links for QEMU}{Useful External Links for QEMU}}\label{md__arduino_2_get_started_with_machine_learning_on_arduino_2tflite-micro-main_2tensorflow_2lite_2micro_2docs_2qemu_autotoc_md224}
The current QEMU implementation uses {\ttfamily user} mode. The documentation for \href{https://www.qemu.org/docs/master/user/index.html}{\texttt{ user mode is here}}.

QEMU uses ARM \href{https://github.com/ARM-software/abi-aa/blob/main/semihosting/semihosting.rst}{\texttt{ semihosting}} to replace newlib system calls for specific boards with the host OS.
