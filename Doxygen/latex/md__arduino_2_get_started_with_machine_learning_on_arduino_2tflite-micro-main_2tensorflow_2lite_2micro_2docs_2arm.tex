Arm\textquotesingle{}s Cortex-\/M processor support is fully integrated to TFLM. To build a TFLM library for any Cortex-\/M processor, check out the \href{https://github.com/tensorflow/tflite-micro/tree/main/tensorflow/lite/micro/cortex_m_generic/README.md}{\texttt{ Cortex-\/M generic readme}}. Additionally, CMSIS-\/\+NN provides optimal performance executing machine learning workloads on Cortex-\/M. See the sub chapter CMSIS-\/\+NN.\hypertarget{md__arduino_2_get_started_with_machine_learning_on_arduino_2tflite-micro-main_2tensorflow_2lite_2micro_2docs_2arm_autotoc_md153}{}\doxysubsection{\texorpdfstring{CMSIS-\/\+NN optimized library}{CMSIS-\/\+NN optimized library}}\label{md__arduino_2_get_started_with_machine_learning_on_arduino_2tflite-micro-main_2tensorflow_2lite_2micro_2docs_2arm_autotoc_md153}
Common Microcontroller Software Interface Standard for Neural Networks (CMSIS-\/\+NN) is a collection of efficient neural network kernels developed to maximize performance on Cortex-\/M processors. The CMSIS-\/\+NN optimized kernel are highly integrated to TFLM. For more information how to utilize these kernels, see \href{https://github.com/tensorflow/tflite-micro/tree/main/tensorflow/lite/micro/kernels/cmsis_nn/README.md}{\texttt{ CMSIS-\/\+NN readme}}.\hypertarget{md__arduino_2_get_started_with_machine_learning_on_arduino_2tflite-micro-main_2tensorflow_2lite_2micro_2docs_2arm_autotoc_md154}{}\doxysubsection{\texorpdfstring{Arm(\+R) Ethos(\+TM)-\/U micro\+NPU family}{Arm(\+R) Ethos(\+TM)-\/U micro\+NPU family}}\label{md__arduino_2_get_started_with_machine_learning_on_arduino_2tflite-micro-main_2tensorflow_2lite_2micro_2docs_2arm_autotoc_md154}
The Ethos-\/U micro\+NPU (Neural Processing Unit) family consist of \href{https://www.arm.com/products/silicon-ip-cpu/ethos/ethos-u55}{\texttt{ Ethos-\/\+U55}} and \href{https://www.arm.com/products/silicon-ip-cpu/ethos/ethos-u65}{\texttt{ Ethos-\/\+U65}}. Ethos-\/\+U55 is designed to accelerate ML inference in area-\/constrained embedded and IoT devices, whereas Ethos-\/\+U65 extends its applicability to be used as an Cortex-\/M subsystem to a larger Arm Cortex-\/A, Cortex-\/R and Neoverse-\/based system.

To get started with TFLM and Ethos-\/U, see the \href{https://github.com/tensorflow/tflite-micro/tree/main/tensorflow/lite/micro/kernels/ethos_u/README.md}{\texttt{ Ethos-\/U readme}}.\hypertarget{md__arduino_2_get_started_with_machine_learning_on_arduino_2tflite-micro-main_2tensorflow_2lite_2micro_2docs_2arm_autotoc_md155}{}\doxysubsection{\texorpdfstring{Arm(\+R) Corstone(\+TM)-\/300 FVP}{Arm(\+R) Corstone(\+TM)-\/300 FVP}}\label{md__arduino_2_get_started_with_machine_learning_on_arduino_2tflite-micro-main_2tensorflow_2lite_2micro_2docs_2arm_autotoc_md155}
\href{https://developer.arm.com/Processors/Corstone-300}{\texttt{ Corstone-\/300}} is a hardware reference design based on the Arm Cortex-\/\+M55 processor, which integrates the Ethos-\/\+U55 micro\+NPU. The \href{https://developer.arm.com/tools-and-software/open-source-software/arm-platforms-software/arm-ecosystem-fvps}{\texttt{ Corstone-\/300 FVP}} (Fixed Virtual Platform) is a model of the hardware which enables execution of full software stacks ahead of silicon.

To get started with TFLM and Corstone-\/300 FVP, see the \href{https://github.com/tensorflow/tflite-micro/tree/main/tensorflow/lite/micro/cortex_m_corstone_300/README.md}{\texttt{ Corstone-\/300 readme}}.
