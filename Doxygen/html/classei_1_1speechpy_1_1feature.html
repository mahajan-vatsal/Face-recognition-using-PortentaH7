Machine Vision using Portenta H7 2
This project aims to develop a face recognition-based access control system using the Arduino Portenta H7 and Vision Shield, leveraging Edge Impulse for machine learning. The system captures facial images, processes them locally using an AI model deployed on the Portenta H7 and determines access based on authorised personnel.

inlinestatic

Compute the Mel-filterbanks. Each filter will be stored in one rows. The columns correspond to fft bins.

Parameters
filterbanksMatrix of size num_filter * coefficients
num_filterthe number of filters in the filterbank
coefficients(fftpoints//2 + 1)
sampling_freqthe samplerate of the signal we are working with. It affects mel spacing.
low_freqlowest band edge of mel filters, default 0 Hz
high_freqhighest band edge of mel filters, default samplerate / 2
output_transposedIf set to true this will transpose the matrix (memory efficient). This is more efficient than calling this function and then transposing as the latter requires the filterbank to be allocated twice (for a short while).
Returns
EIDSP_OK if OK

inlinestatic

Compute MFCC features from an audio signal.

Parameters
out_featuresUse calculate_mfcc_buffer_size to allocate the right matrix.
signalaudio signal structure from which to compute features. has functions to retrieve data from a signal lazily.
sampling_frequency(int): the sampling frequency of the signal we are working with.
frame_length(float): the length of each frame in seconds. Default is 0.020s
frame_stride(float): the step between successive frames in seconds. Default is 0.01s (means no overlap)
num_cepstral(int): Number of cepstral coefficients.
num_filters(int): the number of filters in the filterbank, default 40.
fft_length(int): number of FFT points. Default is 512.
low_frequency(int): lowest band edge of mel filters. In Hz, default is 0.
high_frequency(int): highest band edge of mel filters. In Hz, default is samplerate/2
dc_eliminationWhether the first dc component should be eliminated or not.
Returns
0 if OK
