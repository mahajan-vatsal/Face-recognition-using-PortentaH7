Detailed Description

Abstract interface that returns TfLiteRegistrations given op codes or custom op names. This is the mechanism that ops being referenced in the flatbuffer model are mapped to executable function pointers (TfLiteRegistrations).

◆ MayContainUserDefinedOps() [1/2]

virtual bool tflite::OpResolver::MayContainUserDefinedOps ( ) const
inlineprivatevirtual

Returns true if this OpResolver may contain any "user defined" ops. By "user defined" ops, we mean any op definitions other than those contained in tflite::ops::builtin::BuiltinOpResolver.

If this method returns true, it doesn't necessarily mean that the OpResolver contains a user-defined op, just that the absence of user-defined ops can't be guaranteed.

Note that "user-defined" ops are not the same as "custom" ops; BuiltinOpResolver may support certain "custom" ops, in addition to "builtin" ops, and may not support all of the "builtin" op enum values.
